# DOTA2主题一页挂机放置小游戏美术资源单

## 一、文档说明

本文件根据仓库内《策划文档.md》整理，目标是把当前第一版原型所需的美术资源整理成可执行、可分发、可持续补充的正式资源表。

本项目当前定位为：

- 单场景、一页式挂机放置小游戏
- 主题包装为 DOTA2 炼金术士比赛成长
- 表现核心不是战斗动画，而是页面结构、角色主体、数值状态、技能与装备解锁反馈、日志播报感

因此本资源单的重点不是做大量场景演出，而是优先保证：

1. 页面结构清楚
2. 炼金术士主题明确
3. 策略切换、升级、装备解锁、大招开启、基地推进这些关键状态清楚
4. Godot 接入成本低，适合先做出完整可玩原型

---

## 二、统一美术规范

| 项目 | 规范 |
|---|---|
| 整体风格 | DOTA2 炼金术士主题包装 + 简洁 2D UI 原型风格 |
| 目标感受 | 像在看一场炼金术士比赛成长过程，而不是传统 MOBA 实时操作界面 |
| 页面结构 | 单页面展示，左右为状态与装备，中间为角色主体，上下为比赛信息与日志/技能区 |
| 重点表现 | 角色主体、基地血条、策略切换、技能选择、装备成型、日志播报 |
| 非重点表现 | 地图移动、复杂战斗动画、敌方英雄实体演出 |
| 背景图规格建议 | 1920x1080，PNG |
| 角色主体规格建议 | 透明背景 PNG，高度建议 900~1200 px |
| 图标规格建议 | 256x256，PNG |
| 小图标规格建议 | 128x128，PNG |
| UI 面板/按钮/槽位 | 透明背景 PNG，尽量支持九宫格拉伸 |
| 资源命名 | 统一使用 asset_id，英文小写，下划线分隔 |

## 五、使用建议

1. 第一版先按 P0 接入，确保页面能完整跑通。
2. 第二步补 P1，把“可选、高亮、冷却、爆发、关键事件”这些状态做清楚。
3. 如果人手紧张，飘字类资源可先交给程序用字体样式实现，不必全部先画成图片。
4. 如果后续要进入 AI 批量生图或外包流程，可在本文件基础上继续补充：
   - 每个 asset_id 的参考图
   - 关键词模板
   - 正反向提示词
   - 是否允许程序改色
