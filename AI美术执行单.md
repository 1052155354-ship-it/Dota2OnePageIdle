# DOTA2主题一页挂机放置小游戏 AI美术执行单

## 一、文档说明

本文件用于把《美术资源单.md》中的资源，进一步转成可直接交给 AI 文生图模型执行的标准化描述。

本文件的目标不是讨论美术理论，而是让执行者能够直接复制提示词去出图，并且让后续批量扩写时保持统一结构。

本项目当前美术执行原则：

- 优先服务 Godot 原型接入
- 优先服务单页面挂机放置玩法
- 优先保证资源可读性，而不是追求复杂插画感
- 优先保证角色、图标、面板、状态的功能明确
- 所有资源都要避免做成“海报”而要做成“游戏资源”

---

## 二、适用范围

本执行单当前主要适用于以下类型资源：

1. 角色立绘
2. 背景图
3. 图标
4. UI 面板与按钮
5. 状态特效图

其中最适合直接用 AI 文生图处理的是：

- 角色立绘
- 背景图
- 图标

UI 面板、按钮、槽位类资源也可以交给 AI 先出概念草图，但最终通常仍需人工或程序进一步整理成可用 UI 资源。

---

## 三、推荐使用方式

### 1. 推荐执行流程

资源单 asset_id → 写成 AI美术执行单 → 复制提示词到生图模型 → 出第一版图 → 回看是否符合游戏用途 → 再迭代收紧

### 2. 每次执行时优先检查的四件事

1. 像不像游戏资源，而不是海报
2. 缩小后清不清楚
3. 能不能直接接到 Godot 页面里
4. 有没有偏离本项目“单页面挂机成长”的用途

### 3. 通用原则

- 角色图尽量透明背景
- 背景图尽量单独出，不和角色混画
- 图标尽量单体、居中、边缘清楚
- 避免复杂透视
- 避免大段场景叙事
- 避免模型自由发挥出一堆无关元素

## 四、单个资源执行单标准模板

以下模板用于后续继续扩写所有 asset_id：

```text
【asset_id】

【资源名】

【资源用途】
说明这张图在游戏里放哪里，用来干什么。

【输出目标】
说明最终要得到什么类型的资源图，例如角色立绘、背景图、图标、按钮草图等。

【完整提示词】
直接给生图模型的完整描述。

【负面约束】
告诉模型不要出现什么。

【输出要求】
尺寸、背景、格式、主体是否完整、是否需要透明背景。

【验收重点】
检查这张图是否符合游戏用途。
```

## 五、执行单示例

## 5.1 角色主立绘执行单

### 【asset_id】
char_alchemist_main

### 【资源名】
炼金术士主立绘

### 【资源用途】
用于单页面挂机放置游戏主界面中央角色主体。该角色会长期停留在页面中央，用于表现“炼金术士比赛成长”的核心视觉。它不是战斗动画素材，而是主界面的长期展示资源，因此最重要的是辨识度、轮廓清楚、主题明确、适合和周围 UI 共存。

### 【输出目标】
一张 2D 游戏角色立绘，透明背景 PNG，适合直接放在 Godot 主界面中央。主体完整、边缘干净、缩小后也能清楚识别。

### 【完整提示词】
一张 2D 游戏角色立绘，主题为炼金术士风格英雄，灵感来自 DOTA 气质但不要直接照搬原作设计。角色应表现出强烈的炼金、药剂、毒雾、财富、爆发成长的感觉。整体是一个偏疯狂、贪财、危险、带有实验感的英雄形象，能够让玩家一眼联想到“炼金术士”主题。角色为单人主体，站立待机姿态，身体朝向略微右前方，动作稳定，不做攻击动作，不做施法动作，不做大幅夸张透视。角色轮廓必须清楚，缩小后依然容易识别，适合作为单页面挂机游戏中央长期展示的主视觉资源。

画面风格为 2D 游戏美术，风格化手绘，清晰剪影，大色块明确，具有游戏资源感，而不是海报感。整体气质偏 DOTA 风格包装，但简化为适合独立游戏原型与 Godot 接入的资源风格。不要做成影视概念图，不要做成写实照片，不要做成 3D 渲染，不要做成纯海报，不要做成复杂背景插画。

角色颜色以黄绿色、毒绿色、暗金色、少量金属灰为主，体现炼金、毒液、金币和暴富感。可以加入少量炼金烟雾、药剂蒸汽、绿色能量、金币元素作为辅助视觉，但这些元素只能围绕角色轻微点缀，不能遮挡角色主体，也不能让画面变乱。角色边缘要干净，主体完整，居中清楚，透明背景，没有地面，没有场景，没有 UI，没有文字，没有边框，没有水印。

要求输出为透明背景 PNG 用途的角色资源图。角色完整显示，主体不要被裁切，不要多角色，不要杂乱小物件堆满周围，不要让特效抢走主体视觉。整体目标是让这张图能直接被放进 Godot 的主界面中央，成为一眼可认出的炼金术士主题主立绘。

### 【负面约束】
不要背景场景，不要战场，不要地图，不要建筑，不要 UI，不要文字，不要字幕，不要海报排版，不要电影光影，不要写实照片，不要 3D 渲染，不要模糊，不要低清晰度，不要多角色，不要极端透视，不要过多特效遮挡，不要把角色裁切掉，不要复杂构图，不要把画面做成插画海报而不是游戏资源。

### 【输出要求】
- 透明背景
- PNG
- 单人主体
- 主体完整显示
- 适合 Godot 接入
- 角色居中或近居中
- 缩小后保持清晰轮廓

### 【验收重点】
1. 第一眼能否看出这是炼金术士主题角色。
2. 主体是否清楚，缩小后是否还能辨认。
3. 有没有被模型画成海报、插画或场景图。
4. 是否适合直接摆在主界面中央长期展示。
5. 是否存在多余背景、文字、水印、复杂场景等无关内容。

## 六、当前使用建议

1. 先用本执行单生成 `char_alchemist_main` 第一张测试图。
2. 测试图如果偏海报，就继续收紧“透明背景、游戏资源、不要背景、不要海报感”。
3. 测试图如果太像 DOTA 原版，就继续收紧“只保留炼金术士气质，不直接照搬原作设计”。
4. 等角色主立绘稳定后，再继续扩写：
   - `bg_match_main`
   - `obj_enemy_base_main`
   - 六个装备图标
   - 四个技能图标
